# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any 1Platform repository, please report it privately to **security@1platform.pro**.

- Do **not** open a public issue.
- Include a clear description, reproduction steps, impact assessment, and any proof-of-concept.
- We acknowledge reports within **48 hours** and aim to provide a remediation plan within **7 days**.

## Supported Versions

We support the latest release of each public repository. Older versions may receive security patches at our discretion.

## Scope

In scope:
- `1platformlabs/1platform-content-ai` (WordPress plugin)
- `1platformlabs/1platform-website`
- `1platformlabs/1platform-api-developer`
- Production services running on `*.1platform.pro`

Out of scope:
- Third-party dependencies (report to their maintainers)
- Self-hosted or forked deployments of our open-source code
- Rate-limiting, brute-force, or denial-of-service testing against production

## Responsible Disclosure

We credit researchers who report valid vulnerabilities in our security advisories (opt-in). We do not currently run a paid bug bounty program.
